# TransSuccess 人工测试指南

## 测试环境准备

1. **确认开发环境**
   - 确保已安装 Delphi IDE（推荐使用 Delphi 10.4 或更高版本）
   - 确认 SynEdit 组件已正确安装（路径：D:\ProgramData\SynEdit-master\Source）
   - 确认 JCL 库已正确安装并配置

2. **项目文件检查**
   - 打开 TransSuccess.dpr 主项目文件
   - 检查所有引用的单元是否存在

## 测试项目 1：编译测试

1. **编译主程序**
   - 打开命令提示符或 PowerShell
   - 切换到项目目录：`cd D:\SynologyDrive\Progs\_Delphi\TransSuccess`
   - 执行编译命令：`dcc32 TransSuccess.dpr`
   - 确认编译成功，没有错误或警告

2. **编译测试程序**
   - 编译编码检测器测试：`dcc32 TestEncodingDetectorRunner.dpr`
   - 编译编码转换器测试：`dcc32 TestEncodingConverterRunner.dpr`
   - 编译性能测试：`dcc32 TestEncodingPerformanceRunner.dpr`
   - 编译内存测试：`dcc32 TestEncodingMemoryRunner.dpr`
   - 确认所有测试程序都能成功编译

## 测试项目 2：BOM 检测测试

1. **准备测试文件**
   - 创建一个包含 UTF-8 BOM 的文本文件（使用记事本保存为 UTF-8 格式）
   - 创建一个不包含 BOM 的 UTF-8 文本文件
   - 创建一个 UTF-16LE 文件（包含 BOM）
   - 创建一个 UTF-16BE 文件（包含 BOM）

2. **运行主程序进行测试**
   - 启动编译好的 TransSuccess.exe
   - 使用"打开文件"功能，分别打开上述测试文件
   - 验证程序能正确识别文件的 BOM 和编码
   - 记录每个文件的检测结果

## 测试项目 3：中文编码检测测试

1. **准备中文测试文件**
   - 创建一个 GBK 编码的纯中文文件
   - 创建一个 GB18030 编码的中文文件（包含扩展汉字）
   - 创建一个 Big5 编码的繁体中文文件
   - 创建一个混合中英文的 GBK 文件

2. **运行主程序进行测试**
   - 启动 TransSuccess.exe
   - 使用"分析文件"功能，分别分析上述中文文件
   - 验证是否能正确识别各种中文编码
   - 记录检测置信度和详细分析结果

## 测试项目 4：UTF-8 检测优化测试

1. **准备 UTF-8 测试文件**
   - 创建一个标准 UTF-8 文件（包含多语言内容）
   - 创建一个包含无效 UTF-8 序列的文件
   - 创建一个包含混合内容的 UTF-8 文件（中文、英文、特殊字符）

2. **运行检测测试**
   - 启动 TransSuccess.exe
   - 使用"检测编码"功能分析上述文件
   - 验证程序是否能正确识别 UTF-8 编码并指出潜在问题
   - 测试程序处理无效 UTF-8 序列的能力

## 测试项目 5：编码转换测试

1. **准备转换测试文件**
   - 准备一个 GBK 编码的中文文件
   - 准备一个 Big5 编码的繁体中文文件
   - 准备一个 UTF-16LE 编码的文件

2. **进行编码转换测试**
   - 启动 TransSuccess.exe
   - 分别将上述文件转换为 UTF-8 编码
   - 检查转换后的文件内容是否正确
   - 验证 BOM 处理选项（添加/移除 BOM）是否生效

3. **批量转换测试**
   - 准备一个包含多种编码文件的文件夹
   - 使用"批量转换"功能将所有文件转换为相同编码
   - 验证所有文件是否都被正确转换

## 测试项目 6：错误处理测试

1. **无法识别的编码测试**
   - 准备一个二进制文件（非文本文件）
   - 使用程序尝试检测其编码
   - 验证程序是否给出合理的错误提示而不是崩溃

2. **转换错误处理测试**
   - 准备一个包含无法转换字符的文件
   - 尝试将其转换为不支持这些字符的编码
   - 验证程序是否提供错误处理选项或合理提示

3. **大文件处理测试**
   - 准备一个超过 100MB 的大文本文件
   - 测试程序对其进行编码检测和转换
   - 观察内存使用情况和处理速度

## 测试项目 7：性能测试

1. **批量文件性能测试**
   - 准备一个包含多种编码、多种大小的文件集合（至少 100 个文件）
   - 使用程序对所有文件进行批量编码检测
   - 记录总耗时和平均每个文件的检测时间

2. **大文件性能测试**
   - 分别准备 10MB、50MB、100MB 大小的文本文件
   - 测试每个文件的编码检测和转换时间
   - 记录不同大小文件的处理性能差异

## 测试项目 8：界面功能测试

1. **编辑器功能测试**
   - 测试内置文本编辑器的基本功能
   - 验证编辑器对不同编码文件的显示是否正确
   - 测试编辑后保存功能是否正确维持编码

2. **设置功能测试**
   - 测试编码检测设置是否生效
   - 测试转换选项设置是否生效
   - 测试界面布局设置是否正确保存和恢复
