## 检测算法说明

### 1. BOM检测算法

BOM (Byte Order Mark) 检测是最准确的编码检测方法，通过检查文件开头的特定字节序列来确定编码类型。

- UTF-8 BOM: EF BB BF
- UTF-16LE BOM: FF FE
- UTF-16BE BOM: FE FF
- UTF-32LE BOM: FF FE 00 00
- UTF-32BE BOM: 00 00 FE FF

### 2. UTF-8检测算法

UTF-8无BOM文件的检测基于以下几个方面：

1. **字节序列验证**：检查字节序列是否符合UTF-8编码规则
   - 单字节字符: 0xxxxxxx
   - 双字节字符: 110xxxxx 10xxxxxx
   - 三字节字符: 1110xxxx 10xxxxxx 10xxxxxx
   - 四字节字符: 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx

2. **字节分布统计**：分析高位字节和后续字节的分布情况

3. **中文UTF-8特征分析**：检测常见中文UTF-8编码模式

4. **连续有效序列分析**：检测连续有效的UTF-8序列长度

### 3. 中文编码检测算法

中文编码检测基于以下几个方面：

1. **字节序列验证**：检查字节序列是否符合特定中文编码规则

2. **字节频率分析**：分析字节频率分布，与已知的中文编码特征比较

3. **常见汉字检测**：检测常见汉字在不同编码中的特征

4. **编码特征得分**：综合计算各种编码的特征得分

## 检测准确率影响因素

1. **文件内容**：纯文本文件检测准确率高于二进制文件或混合内容文件

2. **文件大小**：较大文件的检测准确率通常高于小文件

3. **语言混合**：多语言混合文件的检测准确率较低

4. **编码相似性**：某些编码（如GBK和GB18030）有相似的特征，可能导致误判

## 改进方向

1. **机器学习算法**：引入机器学习算法提高编码检测准确率

2. **上下文分析**：考虑文件类型和内容上下文进行编码检测

3. **语言识别**：先识别文件语言，再针对特定语言选择合适的编码检测算法

4. **混合编码处理**：改进对混合编码文件的检测能力

5. **用户反馈机制**：添加用户反馈机制，持续优化检测模型
