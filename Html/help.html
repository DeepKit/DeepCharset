入门指南

系统要求

  • 操作系统：Windows 7/8/10/11（32位或64位）
  • 处理器：1 GHz 或更快
  • 内存：512 MB RAM（推荐 1 GB 或更多）
  • 硬盘空间：50 MB 可用空间
  • 显示器：分辨率 1024x768 或更高

安装步骤

  1. 下载最新版本的 TransSuccess 压缩包
  2. 解压缩到您选择的目录
  3. 运行 TransSuccess.exe 启动程序

TransSuccess 是绿色软件，无需安装，解压后即可使用。

界面概览

TransSuccess 的主界面包含以下几个主要部分：

  • 菜单栏：包含文件、编辑、视图、工具和帮助等菜单
  • 工具栏：常用操作的快捷按钮
  • 文件浏览区：用于选择和浏览文件
  • 文件列表：显示选中目录中的文件
  • 编码选项区：设置源编码和目标编码
  • 状态栏：显示当前状态和操作信息

基本操作

选择文件

有多种方式可以选择要转换的文件：

  1. 使用左侧的目录树浏览并选择文件夹
  2. 在文件列表中选择单个或多个文件（按住 Ctrl 键可多选）
  3. 通过菜单栏的"文件 > 打开"选择文件
  4. 直接将文件拖放到程序窗口中

检测文件编码

选择文件后，TransSuccess 会自动检测文件的编码格式。您可以：

  • 在文件列表中查看检测到的编码
  • 点击"检测编码"按钮重新检测选中文件的编码
  • 双击文件可以预览文件内容，确认编码是否正确

如果自动检测的编码不正确，您可以手动选择正确的源编码。

转换单个文件

  1. 选择要转换的文件
  2. 在"目标编码"下拉列表中选择目标编码格式
  3. 选择是否添加 BOM（字节顺序标记）
  4. 点击"转换"按钮开始转换
  5. 转换完成后，会显示转换结果

预览文件内容

TransSuccess 允许您在转换前预览文件内容，确保编码正确：

  1. 双击文件列表中的文件
  2. 或选中文件后点击"预览"按钮
  3. 在预览窗口中查看文件内容
  4. 可以尝试不同的编码格式，查看哪种编码显示正确

批量转换

选择多个文件

批量转换前，您需要选择多个文件：

  • 在文件列表中按住 Ctrl 键点击多个文件进行选择
  • 按住 Shift 键可以选择连续的文件
  • 使用"编辑 > 全选"或按 Ctrl+A 选择当前目录下的所有文件
  • 使用文件类型过滤器可以只显示特定类型的文件

批量转换设置

批量转换时，您可以设置以下选项：

  • 目标编码：所有文件将转换为此编码
  • 添加 BOM：是否为转换后的文件添加 BOM
  • 创建备份：是否在转换前创建原文件的备份
  • 包含子目录：是否处理子目录中的文件
  • 按文件类型过滤：只转换特定类型的文件

执行批量转换

  1. 选择要转换的多个文件
  2. 设置目标编码和其他选项
  3. 点击"批量转换"按钮
  4. 确认转换操作
  5. 等待转换完成
  6. 查看转换报告

批量转换可能会修改大量文件，建议在转换前创建备份。

转换报告

批量转换完成后，TransSuccess 会生成转换报告，包含以下信息：

  • 转换的文件总数
  • 成功转换的文件数
  • 失败的文件数及原因
  • 每个文件的详细转换信息
  • 转换耗时

您可以保存转换报告，以便日后参考。

高级功能

SVG 转 ICO

TransSuccess 支持将 SVG 矢量图形转换为 ICO 图标文件：

  1. 选择一个 SVG 文件
  2. 点击"SVG转ICON"按钮
  3. 选择保存 ICO 文件的位置
  4. 等待转换完成

转换后的 ICO 文件包含多种尺寸的图标（16x16、32x32、48x48、64x64），适合用作应用程序图标。

文件比较

您可以比较两个文件的内容，查看它们的差异：

  1. 选择第一个文件
  2. 点击"比较"按钮
  3. 选择第二个文件
  4. 查看文件差异

命令行支持

TransSuccess 提供命令行接口，支持脚本和自动化操作：

TransSuccessCLI.exe -i input.txt -o output.txt -e utf-8 -b

更多命令行参数和用法，请参考命令行使用部分。

自定义设置

您可以通过"工具 > 选项"菜单访问设置对话框，自定义以下选项：

  • 界面语言
  • 默认编码格式
  • 文件类型关联
  • 备份设置
  • 预览设置
  • 高级选项

常见问题解决

文件无法打开

如果文件无法打开，可能是由于以下原因：

  • 文件被其他程序锁定
  • 没有足够的权限访问文件
  • 文件路径过长
  • 文件已损坏

解决方法：

  • 关闭可能正在使用该文件的其他程序
  • 以管理员身份运行 TransSuccess
  • 将文件移动到路径较短的位置
  • 检查文件是否完整

编码检测不准确

自动编码检测可能不总是 100% 准确，特别是对于某些特殊文件：

  • 包含多种语言的文件
  • 内容较少的文件
  • 使用非标准编码的文件

解决方法：

  • 手动选择正确的源编码
  • 使用预览功能查看不同编码下的文件内容
  • 尝试不同的编码，直到找到正确的显示

转换后文件出现乱码

如果转换后的文件出现乱码，可能是由于：

  • 源编码选择错误
  • 文件包含无法在目标编码中表示的字符
  • 文件已损坏

解决方法：

  • 确保正确选择源编码
  • 选择支持更广泛字符集的目标编码（如 UTF-8）
  • 使用备份文件重新尝试转换

程序运行缓慢

如果程序运行缓慢，可能是由于：

  • 处理大量文件
  • 文件体积过大
  • 系统资源不足

解决方法：

  • 减少同时处理的文件数量
  • 关闭其他占用资源的程序
  • 升级系统配置

命令行使用

基本语法

TransSuccessCLI.exe [选项] [文件...]

常用选项

  • -i, --input <file> - 指定输入文件
  • -o, --output <file> - 指定输出文件
  • -e, --encoding <encoding> - 指定目标编码
  • -s, --source-encoding <encoding> - 指定源编码
  • -b, --bom - 添加 BOM
  • -n, --no-bom - 不添加 BOM
  • -r, --recursive - 递归处理子目录
  • -f, --filter <pattern> - 文件过滤器
  • -v, --verbose - 显示详细信息
  • -h, --help - 显示帮助信息

示例

转换单个文件：

TransSuccessCLI.exe -i input.txt -o output.txt -e utf-8 -b

批量转换目录中的所有文本文件：

TransSuccessCLI.exe -e utf-8 -r -f "*.txt" C:\MyFiles

转换文件并指定源编码：

TransSuccessCLI.exe -i input.txt -o output.txt -s gbk -e utf-8 -n

命令行模式适合集成到脚本或自动化流程中，实现批量转换。
