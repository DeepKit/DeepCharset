# TransSuccess 快速启动指南

## 程序启动与性能诊断

### 启动时间优化

TransSuccess 已针对启动性能进行优化。首屏显示包含以下阶段：

1. **主窗体初始化** - UI 组件加载
2. **界面设置加载** - 从 `ini/ui.ini` 读取用户偏好  
3. **编码列表准备** - 首次点击时才真正构建（Lazy Load）
4. **语言选择器创建** - 首次下拉时加载实际语言列表
5. **启动完成** - 程序即刻可用，文件列表等待用户操作

启动通常在 **1-2 秒** 内完成。

### 诊断启动瓶颈（Probe 日志）

若启动仍感觉缓慢，可查看程序日志中的 **[Probe]** 记录来定位耗时阶段：

```
[hh:nn:ss.zzz] [Probe] InitStringGrid: +X ms, total Y ms
[hh:nn:ss.zzz] [Probe] LoadUiSettings: +X ms, total Y ms
[hh:nn:ss.zzz] [Probe] CreateLanguageSelector: +X ms, total Y ms
[hh:nn:ss.zzz] [Probe] EndInitializeUI: +X ms, total Y ms
```

- **+X ms** = 该阶段耗时
- **total Y ms** = 累计启动耗时

若某个阶段明显偏慢（>500ms），请截图或复制日志行发送反馈。

## 文件扫描与编码检测

### 扫描与检测工作流

1. 选择目录 → 点击"刷新"按钮
2. **扫描阶段**：递归查找选中扩展名的文件，显示进度条
3. **检测阶段**：后台线程批量检测文件编码，结果逐批投递更新网格

扫描完成后，网格左上方会显示"**检测统计**"日志：
```
[hh:nn:ss.zzz] 检测统计: 已入队=1234, 已投递(批)= 1234
```

- **已入队** = 发现并加入检测队列的文件总数
- **已投递(批)** = 已完成检测且更新到网格的文件总数

若两数字相差较大，说明仍在进行检测，稍等片刻即可完成。

### 编码检测参数调优

若希望更快获得首批检测结果或减少网格刷新频率，可编辑 `ini/ui.ini`：

```ini
[Detection]
BatchSize=64
FlushIntervalMS=200
```

**参数说明：**
- **BatchSize** (范围 1–1024，默认 64)
  - 单次批量的最大检测文件数
  - 减小该值 → 更频繁地更新网格，反馈更快；但主线程压力增加
  - 增大该值 → 每次批量更大，更新间隔更长；较适合大目录

- **FlushIntervalMS** (范围 20–5000，默认 200)
  - 批量最久刷新间隔（毫秒）
  - 若未满批，超过此时间也会投递
  - 减小该值 → 反馈更及时；但更新更频繁
  - 增大该值 → 较少打扰 UI，但等待时间更长

**推荐配置：**
- **小目录/快反馈**：BatchSize=16, FlushIntervalMS=100
- **大目录/低压力**：BatchSize=128, FlushIntervalMS=400
- **默认（平衡）**：BatchSize=64, FlushIntervalMS=200

## 性能数据示例

### 小目录（<100 文件）
- 扫描 + 检测：~300-500ms
- Probe 示例：
  ```
  [Probe] InitStringGrid: +25 ms, total 25 ms
  [Probe] LoadUiSettings: +30 ms, total 55 ms
  [Probe] CreateLanguageSelector: +10 ms, total 65 ms
  [Probe] EndInitializeUI: +5 ms, total 70 ms
  ```

### 大目录（>5000 文件）
- 扫描：~1-3s（包括目录遍历）
- 检测：分批进行，首批结果通常在 500-1000ms 内出现
- 完整检测可能需要数秒，但网格会实时更新反馈进度

## 常见问题

### Q: 扫描完成后仍看到"..."（编码检测中占位符）
**A:** 这是正常现象。检测在后台进行，网格会逐批更新编码列。通常数秒内完成。若长时间停留，请检查 `[Detection]` 参数或系统资源占用。

### Q: 如何加快编码检测？
**A:** 
1. 减小 BatchSize（如 32）以获得更及时反馈
2. 减小 FlushIntervalMS（如 100）缩短刷新间隔
3. 确保选中的扩展名数量不过多
4. 在系统不忙时进行扫描

### Q: 大目录扫描时程序无响应？
**A:** 正常现象。扫描和检测在后台线程运行，主线程仍可交互。若需取消，点击"取消"按钮即可立即停止。

1. **Probe 日志**（从程序日志窗口复制）

3. **检测统计**（已入队 vs 已投递数字）
4. **系统配置**（CPU、内存、磁盘类型）
